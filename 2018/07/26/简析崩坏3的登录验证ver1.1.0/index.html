简析崩坏3的登录验证（ver1.1.0）

发表于 2018-07-26 | | 阅读次数:

在ver1.1.0版本中，如果修改了apk中的任意文件，都会导致游戏登录时提示“不是官方的apk”，并要求重新下载游戏。下文中将对验证机制做简略分析，并提出两种绕过登录验证的方式。

0x03 JAVA层代码分析

由dll代码分析可知check_sum应为apkCommentInfo.checksum+“f7810749_4fee4fe9e60c3370”，check_sum中的未知值与android_signature的获取应在java层代码中有相关实现。利用androidkiller对apk进行反编译，得到classes-dex2jar.jar，用jd-gui打开后定位到com.miHoYo.ApkCommentReader与com.miHoYo.MihoyoAndroidUtil。com.miHoYo.ApkCommentReader中的getApkComment()用于获取并返回apk zip包的注释信息（cps xor 0xcc）。isMihoyoComment()用于判断apk内容是否被篡改，若被篡改，返回false，反之为true。

com.miHoYo.MihoyoAndroidUtil中的getMetaInfCheckSum()用于获取证书MANIFEST.MF的crc32+all MD5，getSign()用于获取当前apksignature的hashcode，通过编写Xposed插件hook上述两个函数并打印出log，得到
